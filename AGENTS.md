# AGENTS.md

- 把本文件当作索引，不要把所有细节都堆在这里。
- 更完整的协作规范以 `docs/` 下文档为准。
- 每次开发任务开始前，都要先按下面的阅读顺序获取上下文。

提交前检查:
- [PR 模板与检查清单](D:/桌面文件/project/知识库agent/docs/双端协同统一手册.md#14-pr-模板与检查清单)

给 Codex / Claude Code 下发任务时可参考:
- [提示词模板](D:/桌面文件/project/知识库agent/docs/双端协同统一手册.md#13-提示词模板)

## 2. 每次开发的默认流程

1. 先明确目标、非目标、允许修改范围、禁止修改范围。
2. 再冻结接口、数据结构、配置项、错误语义等契约。
3. 再搜索现有实现，优先复用已有模式，不凭空重写。
4. 开始改代码前，先确认本轮需要验证的命令或检查项。
5. 完成修改后，做最小必要验证，并明确未验证项和风险。

## 3. 输出要求

完成任务时默认输出:
- 已修改文件
- 核心改动
- 已执行验证
- 未执行验证
- 已知风险或待确认项

## 4. 仓库约定

- 如果需要新增长期规则，优先更新 `docs/`，再回链本文件。
- 如果某个子目录后续需要更细的局部规范，可在子目录放置更深层的 `AGENTS.md`。
