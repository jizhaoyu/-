# 从零到一执行清单

适用项目: `knowledge-base-agent`

目标:
- 跑起 Spring Boot 服务
- 打开内置前端页面
- 使用本机 MySQL 和 Redis
- 使用 Docker 启动 Milvus 依赖
- 完成上传文档 -> 建索引 -> 问答 的完整链路

## 1. 基础环境

- [ ] 安装 `JDK 17`
- [ ] 安装 `Maven 3.9+`
- [ ] 确认本机已有 `MySQL 8`
- [ ] 确认本机已有 `Redis`
- [ ] 安装 `Docker Desktop`

说明:
- 这个项目没有单独的前端工程。
- 页面由 Spring Boot 直接托管，启动后访问 `/` 即可。
- 本项目当前约定:
  - `MySQL` 使用本机服务
  - `Redis` 使用本机服务
  - `Milvus` 使用 `docker compose`

## 2. 获取代码

- [ ] 进入项目根目录 `D:\桌面文件\project\知识库agent`
- [ ] 确认存在 [pom.xml](D:/桌面文件/project/知识库agent/pom.xml)
- [ ] 确认存在 [docker-compose.yml](D:/桌面文件/project/知识库agent/docker-compose.yml)
- [ ] 确认存在 [application.yml](D:/桌面文件/project/知识库agent/src/main/resources/application.yml)

## 3. 本机 MySQL 准备

- [ ] 启动本机 MySQL
- [ ] 创建数据库 `kb_agent`
- [ ] 确认账号密码可用，默认配置为 `root / 123456`

示例:

```sql
CREATE DATABASE IF NOT EXISTS kb_agent
DEFAULT CHARACTER SET utf8mb4
DEFAULT COLLATE utf8mb4_unicode_ci;
```

默认连接配置来自 [application.yml](D:/桌面文件/project/知识库agent/src/main/resources/application.yml):

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/kb_agent?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai&useSSL=false&allowPublicKeyRetrieval=true
    username: root
    password: 123456
```

如果你的本机配置不同，设置环境变量:

- [ ] `MYSQL_URL`
- [ ] `MYSQL_USERNAME`
- [ ] `MYSQL_PASSWORD`

## 4. 本机 Redis 准备

- [ ] 启动本机 Redis
- [ ] 确认 `localhost:6379` 可访问

默认连接配置:

```yaml
spring:
  data:
    redis:
      host: localhost
      port: 6379
```

说明:
- Redis 不是启动阻塞项。
- 即使 Redis 没启动，页面和主链路仍可运行，只是查询缓存会降级。

## 5. AI 配置

要让文档真正完成索引并进入 `READY`，必须提供:

- [ ] `AI_CHAT_API_KEY`
- [ ] `AI_EMBEDDING_API_KEY`

按需覆盖:

- [ ] `AI_CHAT_BASE_URL`
- [ ] `AI_CHAT_MODEL`
- [ ] `AI_EMBEDDING_BASE_URL`
- [ ] `AI_EMBEDDING_MODEL`

PowerShell 示例:

```powershell
$env:AI_CHAT_API_KEY="your-chat-key"
$env:AI_EMBEDDING_API_KEY="your-embedding-key"
```

如果不配置 AI key:

- 文档不会永久卡在 `PROCESSING`
- 文档会转成 `FAILED`
- 常见错误是 `embedding API key is missing`

## 6. 启动最小可运行版本

先跑最小链路，确认页面和接口正常:

- [ ] 使用内存向量库启动项目
- [ ] 打开页面
- [ ] 验证运行态接口

命令:

```powershell
mvn -DskipTests spring-boot:run "-Dspring-boot.run.arguments=--app.milvus.enabled=false"
```

验证地址:

- [ ] `http://localhost:8080/`
- [ ] `http://localhost:8080/api/kb/runtime`

最小模式下预期:

- [ ] `MySQL = UP`
- [ ] `Storage = UP`
- [ ] `Redis = UP` 或 `WARN`
- [ ] `Milvus = DISABLED`
- [ ] `vectorStoreMode = IN_MEMORY`

- [ ] 首页能打开
- [ ] 文档管理区能展示文档列表
- [ ] 运行态面板能显示依赖状态
- [ ] 上传按钮可用
- [ ] 问答区域可输入问题

静态资源应可访问:

- [ ] `http://localhost:8080/app.js`

## 8. 文档上传与索引验收

- [ ] 上传一个 `txt` 文件
- [ ] 文档初始状态显示 `PROCESSING`
- [ ] 等待状态变化

有 AI key 时的预期:

- [ ] 文档最终变为 `READY`

没有 AI key 时的预期:

- [ ] 文档最终变为 `FAILED`
- [ ] 能看到明确错误信息

注意:
- 当前仓库已经修复了“上传后长期卡在 `PROCESSING`”的问题。
- 索引触发现在是事务提交后再执行，见 [DocumentServiceImpl.java](D:/桌面文件/project/知识库agent/src/main/java/com/knowledge/agent/service/impl/DocumentServiceImpl.java:154)

## 9. 问答验收

- [ ] 文档进入 `READY` 后发起一次问答
- [ ] 返回答案
- [ ] 返回引用片段
- [ ] 页面能展示文件名和片段内容

如果知识库为空或未命中:

- 允许返回降级回答

## 10. 启动 Milvus 真实模式

前提:

- [ ] 已安装 Docker
- [ ] `docker` 命令可用

说明:
- 当前 [docker-compose.yml](D:/桌面文件/project/知识库agent/docker-compose.yml) 只负责 `etcd`、`minio`、`milvus`
- 不会重复启动本机 MySQL 和 Redis

在项目根目录执行:

```powershell
docker compose up -d
```

需要确认的服务:

- [ ] `etcd`
- [ ] `minio`
- [ ] `milvus`

需要确认的端口:

- [ ] `19530`（Milvus REST）
- [ ] `9091`（Milvus metrics / health）
- [ ] `9000`
- [ ] `9001`

默认 Milvus 配置:

```yaml
app:
  milvus:
    enabled: false
    base-url: http://localhost:19530
    token: root:Milvus
    collection-name: kb_chunk
    vector-dimension: 1536
```

启用 Milvus 模式:

```powershell
mvn -DskipTests spring-boot:run "-Dspring-boot.run.arguments=--app.milvus.enabled=true"
```

Milvus 模式下预期:

- [ ] `Milvus = UP`
- [ ] `vectorStoreMode = MILVUS`

- [ ] 先确认本机 MySQL 可用

- [ ] 再配 AI key
- [ ] 先用 `IN_MEMORY` 模式跑通页面和上传
- [ ] 再用 Docker 启动 Milvus 并切到 `MILVUS` 模式
- [ ] 最后验证问答和引用
