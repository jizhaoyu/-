# 测试与环境准备

本文档基于当前项目的实际运行方式整理:

- `MySQL` 使用本机服务
- `Redis` 使用本机服务
- `Milvus` 使用 Docker 启动
- 前端页面由 Spring Boot 直接托管，不是独立前端工程

## 1. 环境要求

- `JDK 17`
- `Maven 3.9+`
- 本机可用的 `MySQL 8`
- 本机可用的 `Redis`
- 可用的 `Docker Desktop`

建议:

- 运行测试优先使用 `JDK 17`
- 避免直接用 `JDK 25` 跑 Mockito 测试，可能遇到 agent 兼容问题

## 2. 项目默认端口

- Spring Boot: `8080`
- MySQL: `3306`
- Redis: `6379`
- Milvus REST: `19530`
- Milvus metrics / health: `9091`
- MinIO API: `9000`
- MinIO Console: `9001`

## 3. 本机 MySQL 准备

确认本机 MySQL 正常启动，并创建数据库 `kb_agent`。

示例:

```sql
CREATE DATABASE IF NOT EXISTS kb_agent
DEFAULT CHARACTER SET utf8mb4
DEFAULT COLLATE utf8mb4_unicode_ci;
```

项目默认配置见 [application.yml](D:/桌面文件/project/知识库agent/src/main/resources/application.yml):

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/kb_agent?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai&useSSL=false&allowPublicKeyRetrieval=true
    username: root
    password: 123456
```

如果本机配置不同，请设置环境变量:

```powershell
$env:MYSQL_URL="jdbc:mysql://localhost:3306/kb_agent?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai&useSSL=false&allowPublicKeyRetrieval=true"
$env:MYSQL_USERNAME="root"
$env:MYSQL_PASSWORD="123456"
```

## 4. 本机 Redis 准备

确认本机 Redis 正常启动，并监听 `6379`。

默认配置:

```yaml
spring:
  data:
    redis:
      host: localhost
      port: 6379
```

说明:

- Redis 不是启动阻塞项
- Redis 不可用时，系统会自动降级缓存逻辑
- 页面和主接口仍然可以工作

## 5. AI 配置

如果要让文档真正完成向量化并进入 `READY`，至少需要:

- `AI_CHAT_API_KEY`
- `AI_EMBEDDING_API_KEY`

PowerShell 示例:

```powershell
$env:AI_CHAT_API_KEY="your-chat-key"
$env:AI_EMBEDDING_API_KEY="your-embedding-key"
```

可选覆盖:

```powershell
$env:AI_CHAT_BASE_URL="https://api.moonshot.cn/v1"
$env:AI_CHAT_MODEL="kimi-k2.5"
$env:AI_EMBEDDING_BASE_URL="https://dashscope.aliyuncs.com/compatible-mode/v1"
$env:AI_EMBEDDING_MODEL="text-embedding-v2"
```

如果缺少 AI key:

- 上传文档后不会永久卡在 `PROCESSING`
- 文档会转成 `FAILED`
- 常见报错为 `embedding API key is missing`

## 6. 启动 Milvus 依赖

当前 [docker-compose.yml](D:/桌面文件/project/知识库agent/docker-compose.yml) 只负责:

- `etcd`
- `minio`
- `milvus`

不会重复启动本机 MySQL 和 Redis。

启动命令:

```powershell
docker compose up -d
```

启动后确认:

```powershell
docker compose ps
```

应看到:

- `kb-agent-etcd`
- `kb-agent-minio`
- `kb-agent-milvus`

## 7. 启动项目

### 7.1 local 默认 Milvus 模式

Milvus 依赖启动后，local 环境默认走真实向量库，无需额外传参:

```powershell
mvn -DskipTests spring-boot:run
```

预期:

- `vectorStoreMode = MILVUS`
- `Milvus = UP`

### 7.2 显式回退到内存向量模式

如需只验证最小可运行链路，可显式关闭 Milvus:

```powershell
mvn -DskipTests spring-boot:run "-Dspring-boot.run.arguments=--app.milvus.enabled=false"
```

预期:

- `vectorStoreMode = IN_MEMORY`
- `Milvus = DISABLED`

## 9. 文档上传验证

验证顺序:

1. 上传一个 `txt` 文件
2. 观察文档状态从 `PROCESSING` 开始变化
3. 如果 AI 配置正常，状态应变为 `READY`
4. 如果 AI 配置缺失，状态应变为 `FAILED`

当前仓库已经修复了“上传后长期卡在 `PROCESSING`”的问题，修复位置在 [DocumentServiceImpl.java](D:/桌面文件/project/知识库agent/src/main/java/com/knowledge/agent/service/impl/DocumentServiceImpl.java:154)。

## 10. 问答验证

文档进入 `READY` 后，验证:

- 输入问题
- 返回答案
- 返回引用片段
- 页面展示引用文件名和片段内容

## 11. 常见问题

### 11.1 文档一直失败

优先检查:

- `AI_CHAT_API_KEY`
- `AI_EMBEDDING_API_KEY`
- embedding 模型配置

### 11.2 Runtime 里 Redis 是 WARN

说明 Redis 没连上，但不一定影响主功能。

### 11.3 Runtime 里 Milvus 是 DISABLED

说明当前还在内存向量模式，或者启动命令里使用了:

```powershell
--app.milvus.enabled=false
```

### 11.4 Runtime 里 Milvus 不是 UP

优先检查:

- `docker compose ps`
- `19530` 端口是否可访问
- `APP_MILVUS_BASE_URL` 是否被改错
- `MILVUS_TOKEN` 是否与项目配置一致

## 12. 推荐执行顺序

1. 确认本机 MySQL 可用
2. 确认本机 Redis 可用
3. 配置 AI key
4. 用 `IN_MEMORY` 模式跑通页面和上传
5. 使用 Docker 启动 Milvus
6. 切到 `MILVUS` 模式
7. 验证问答和引用
