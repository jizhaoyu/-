# 文档详情联调接口清单

## 1. 联调目标

点击文档列表中的“详情”按钮后：

1. 先跳转到独立详情页 `/details-{id}`
2. 详情页先显示文档元数据
3. 再异步加载文档正文
4. 正文加载失败时，不影响元数据展示
5. 状态刷新继续只走轻量详情接口，不读取正文

---

## 2. 联调时序

### 2.1 文档列表页加载

前端调用：

```http
GET /api/kb/documents?kbId=default-kb&pageNum=1&pageSize=10
```

用途：
- 渲染文档表格
- 点击“详情”按钮前的数据来源

### 2.2 点击“详情”按钮后，先拉元数据

浏览器先跳转：

```http
GET /details-{id}
```

示例：

```http
GET /details-1001
```

页面用途：
- 独立承载文档详情视图
- 支持浏览器直接打开、刷新和分享链接

进入详情页后，前端再调用：

```http
GET /api/kb/documents/{id}
```

用途：
- 展示状态、标签、时间、错误信息
- 初始化独立详情页的元数据区域
- 先把正文区设置为“加载中”

### 2.3 再异步拉正文

前端调用：

```http
GET /api/kb/documents/{id}/content
```

用途：
- 只更新详情卡中的“文档正文”区域
- 不覆盖已展示的元数据

### 2.4 上传 / 重建索引后继续轮询元数据

前端继续调用：

```http
GET /api/kb/documents/{id}
```

用途：
- 跟踪 `PROCESSING / READY / FAILED`
- 不调用 `/content`
- 避免轮询时重复解析 PDF / TXT 正文

---

## 3. 接口清单

## 3.1 分页查询文档列表

### 请求

```http
GET /api/kb/documents?kbId=default-kb&pageNum=1&pageSize=10
```

### 请求参数

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `kbId` | string | 是 | 知识库 ID |
| `pageNum` | int | 是 | 页码 |
| `pageSize` | int | 是 | 每页条数 |
| `status` | string | 否 | 可选：`PROCESSING` / `READY` / `FAILED` |
| `fileName` | string | 否 | 文件名模糊搜索 |

### 响应示例

```json
{
  "code": 200,
  "message": "ok",
  "data": {
    "records": [
      {
        "documentId": "1001",
        "kbId": "default-kb",
        "fileName": "demo.txt",
        "sourceType": "txt",
        "status": "READY",
        "tags": ["spring", "rag"],
        "errorMessage": null,
        "createdAt": "2026-03-18T14:00:00",
        "updatedAt": "2026-03-18T14:01:00",
        "indexedAt": "2026-03-18T14:01:10"
      }
    ],
    "total": 1,
    "pageNum": 1,
    "pageSize": 10
  }
}
```

### 前端使用字段

- `documentId`
- `fileName`
- `sourceType`
- `status`
- `tags`
- `updatedAt`

---

## 3.2 查询文档元数据 / 状态

### 请求

```http
GET /api/kb/documents/{id}
```

示例：

```http
GET /api/kb/documents/1001
```

### 作用

- 展示详情卡的元数据
- 给状态轮询使用

### 成功响应示例

```json
{
  "code": 200,
  "message": "ok",
  "data": {
    "documentId": "1001",
    "kbId": "default-kb",
    "fileName": "demo.txt",
    "sourceType": "txt",
    "status": "READY",
    "tags": ["spring", "rag"],
    "errorMessage": null,
    "createdAt": "2026-03-18T14:00:00",
    "updatedAt": "2026-03-18T14:01:00",
    "indexedAt": "2026-03-18T14:01:10"
  }
}
```

### 前端展示区域

- 标题：`fileName`
- 状态：`status`
- 文档 ID：`documentId`
- 知识库 ID：`kbId`
- 文件类型：`sourceType`
- 标签：`tags`
- 创建时间：`createdAt`
- 索引时间：`indexedAt`
- 错误信息：`errorMessage`

### 注意

- 这个接口不返回正文
- 这个接口要保持轻量，供轮询使用

---

## 3.3 读取文档正文

### 请求

```http
GET /api/kb/documents/{id}/content
```

示例：

```http
GET /api/kb/documents/1001/content
```

### 作用

- 单独读取并返回文档正文
- 由详情页异步加载

### 成功响应示例

```json
{
  "code": 200,
  "message": "ok",
  "data": {
    "documentId": "1001",
    "content": "这是文档正文。\n支持多行文本。"
  }
}
```

### 前端展示规则

- 使用纯文本展示
- 需要 HTML 转义
- 不做富文本渲染
- 区域限高，可滚动

### 失败时前端行为

- 元数据区域保持不变
- 正文区域显示错误提示，例如：
  - `正文加载失败：parse file failed`

---

## 3.4 重建索引

### 请求

```http
POST /api/kb/reindex/{id}
```

### 成功响应示例

```json
{
  "code": 200,
  "message": "ok",
  "data": {
    "documentId": "1001",
    "status": "PROCESSING",
    "traceId": "trace123"
  }
}
```

### 前端行为

1. 刷新列表
2. 重新加载元数据详情
3. 开始轮询 `/api/kb/documents/{id}`
4. 不要轮询正文接口

---

## 4. 错误码清单

| 场景 | code | message 示例 |
| --- | --- | --- |
| 参数非法 | `400` | `id must be greater than 0` |
| 文档不存在 | `4101` | `document not found: 1001` |
| 文档解析失败 | `4103` | `parse file failed` |
| 系统异常 | `500` | `internal system error` |

---

## 5. 前端联调检查项

### 列表
- [ ] 列表正常展示文档
- [ ] “详情”按钮能拿到正确 `documentId`

### 详情元数据
- [ ] 点击“详情”后先显示元数据
- [ ] 当前行高亮
- [ ] `queryDocId` 同步为当前文档 ID

### 正文
- [ ] 元数据展示后，正文区先显示“加载中”
- [ ] txt 文档能看到正文
- [ ] pdf 文档能看到解析后的文本
- [ ] 含 HTML 标签的正文只显示文本，不执行脚本
- [ ] 正文失败时只影响正文区，不影响元数据区

### 轮询
- [ ] 上传后状态轮询正常
- [ ] 重建索引后状态轮询正常
- [ ] 轮询期间没有反复请求 `/content`

## 6. 推荐联调样例

### 步骤 1：查列表

```http
GET /api/kb/documents?kbId=default-kb&pageNum=1&pageSize=10
```

### 步骤 2：取某条详情

```http
GET /api/kb/documents/1001
```

### 步骤 3：取正文

```http
GET /api/kb/documents/1001/content
```

### 步骤 4：如需重建索引

```http
POST /api/kb/reindex/1001
```

### 步骤 5：轮询状态

```http
GET /api/kb/documents/1001
```
