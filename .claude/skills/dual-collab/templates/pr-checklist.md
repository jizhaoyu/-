# 交付与验收清单

> 用于提交前自检、交接说明或 PR 描述。
> 核心目标是统一验收口径，而不是只听“已经完成”。

## 最小交付要求

交付结果至少应包含：

- 改动文件列表
- 已执行验证
- 未验证项
- 已知风险

## 通用检查项

- [ ] 只提交了与本任务相关的改动
- [ ] 没有覆盖他人未合并工作
- [ ] 公共文件改动已说明影响面
- [ ] 没有提交密钥、token、私密配置
- [ ] 没有提交临时调试代码
- [ ] 已写清接口、配置、数据结构变化
- [ ] 已写清已验证和未验证项
- [ ] 已写清风险

## 后端任务补充检查

- [ ] 接口契约已同步
- [ ] 错误语义没有漂移
- [ ] 至少一个正常场景和一个失败场景已验证
- [ ] 配置缺失时错误清晰
- [ ] 外部依赖失败路径可定位

## 前端任务补充检查

- [ ] Loading、Empty、Error 状态已覆盖
- [ ] 主要交互已验证
- [ ] 接口失败提示合理
- [ ] 移动端或响应式已验证

## 脚本 / 数据任务补充检查

- [ ] 输入输出格式稳定
- [ ] 小样本已跑通
- [ ] 非法输入可处理
- [ ] 写操作不会误覆盖关键数据
- [ ] 失败后可定位、可重跑

## DevOps / 部署任务补充检查

- [ ] 环境变量变化已说明
- [ ] 部署步骤已说明
- [ ] 回滚步骤已说明
- [ ] 健康检查方式已说明

## 推荐评审顺序

1. 先看目标和非目标
2. 再看接口、配置、数据结构是否有变化
3. 再看代码是否覆盖主链路和异常链路
4. 再看验证结果是否足够
5. 最后看文档和回滚说明是否完整
